# 백내장 검사자 리스트 — 컬럼 정의서

> SoftCRM **"백내장 검사자 리스트"** 화면(`FrmCataract_ExamList`)이 추출하는 데이터 ↔ DB 소스 매핑.
> 시력교정 [검사자리스트-컬럼정의.md](검사자리스트-컬럼정의.md)의 **백내장 대응판**. 소스 테이블이 `Cataract_Exam` 계열로 다르다.
> 모든 SQL은 MSSQL(SOFTCRM) 기준, READ-ONLY, `WITH(NOLOCK)` + 날짜 범위 필수.
> 최종 갱신: 2026-06-11 · 상태: **조사 완료(레거시 확인), 구현 전**

## 화면 정체

| 항목 | 값 |
|------|-----|
| 화면명 | **백내장 검사자 리스트** |
| 원본 클래스 | `SoftCrm.FrmCataract_ExamList` (디컴파일 통합본 `SoftCrm.decompiled.cs:745768~749152`) |
| 메뉴 | `통계 > 백내장 검사자관리 > 백내장 검사자 리스트` (코드 A0401) |
| 엑셀/리포트 | 리포트명 `통계(A)-백내장 검사자 리스트`, 엑셀 파일명 `백내장 검사자 리스트{yyMMdd}` |
| 권한 | 통계관리 |
| 뷰 토글 | **radCustom**(고객당 1행) ↔ **radODOS**(좌/우안 각 1행) · **radLast**(최근검사만) ↔ **radTotal**(전체) |
| 기본 필터 | 검사기간(`Exam_Date`), 상담사·상담의·검안사, 유입(InflowCod), Post LK, 수술불가·검사중단 제외 |

> ⚠️ **별개 화면 3종 혼동 주의**
> - `FrmCataract_ExamList` — **백내장 검사자 리스트**(환자 행 목록) ← 본 문서
> - `FrmCataract_ExamCounselList` — 백내장 **상담사별 통계 요약**(집계, 행 목록 아님)
> - `FrmCounselExamList`(시력교정 통합 화면)의 **백내장 UNION 분기** — 25컬럼만, 본 전용 화면(47~49컬럼)보다 빈약
> - `FrmCounselCataractOpList` — 백내장 **수술자** 리스트(검사 아님, `Cataract_Operationdata` 기준)

## 1. 데이터 소스 (쿼리 구조)

디컴파일 문자열결합 SQL을 재구성한 골격 (**radCustom = 고객당 1행** 분기):

```sql
SELECT A.*,                                            -- Cataract_Exam 전체(검사정보·견적·영업가율 등)
   B.OperationR_Date, B.OperationL_Date,               -- 좌/우 수술일
   B.OperationR [OP_OPR], B.OperationL [OP_OPL],       -- 좌/우 수술방법(실제)
   C.emp_name [OperationR_Doc_Name], D.emp_name [OperationL_Doc_Name],  -- 좌/우 집도의
   E.Cust_name, E.Call_Num2, E.Zip_Code, E.Addr1, E.Addr2, E.Etc, E.Job, E.Post_LK,
   ShortJumin [JUMIN_NUM] /* 보안DB면 @SEC_JUMIN */,
   F.emp_name [My_Counselor_Name], G.emp_name [My_Doctor_Name], L.emp_name [My_Optometrist_Name],
   (SELECT NAME FROM MOTIVE_CATEGORY01 WHERE PKEY=K.CATEGORY01_KEY) [CATEGORY01_NAME],   -- 내원동기 대
   (SELECT NAME FROM MOTIVE_CATEGORY02 WHERE PKEY=K.CATEGORY02_KEY) [CATEGORY02_NAME],   -- 중
   (SELECT NAME FROM MOTIVE_CATEGORY03 WHERE PKEY=K.CATEGORY03_KEY) [CATEGORY03_NAME],   -- 세
   (SELECT TOP 1 COMMENTS FROM MOTIVE_NEW01 WHERE CUST_NUM=K.CUST_NUM) [COMMENTS],       -- 동기메모
   (SELECT TOP 1 ... FROM RESERVATION F WHERE RESERVE_FLAG='O' ... ) AS PRE_DATE,        -- 수술예약일
   I.RESERVE_DATE [OP_RESERVE_DATE],
   (SELECT EtcCodVal FROM EtcCfg WHERE EtcBseCod='InflowItemCfg' AND EtcCod=A.InflowCod) [Inflow_Name],  -- 유입
   (SELECT EtcCodNam FROM EtcCfg WHERE EtcBseCod='InsuranceCfg' AND EtcCod=E.InsuranceCode) [InsuranceName], -- 실손
   NC.Nation_CodeName,                                 -- 국적
   TMP2.*                                              -- 가격/수납 집계(아래)
FROM Cataract_Exam A
   LEFT JOIN Cataract_Operationdata B ON A.Cust_num = B.Cust_num
   LEFT JOIN Employee C ON B.OperationR_Doc = C.Emp_num     -- 우안 집도의
   LEFT JOIN Employee D ON B.OperationL_Doc = D.Emp_num     -- 좌안 집도의
   LEFT JOIN Custom   E ON A.Cust_num = E.Cust_num
   LEFT JOIN Employee F ON E.My_Counselor   = F.Emp_num     -- 상담사
   LEFT JOIN Employee G ON E.My_Doctor      = G.Emp_num     -- 상담의
   LEFT JOIN Employee L ON E.My_Optometrist = L.Emp_num     -- 검안사
   LEFT JOIN MOTIVE_NEW02 K ON A.CUST_NUM = K.Cust_num AND K.idx='1'
   LEFT JOIN RESERVATION I (최신 수술예약 O) ...
   LEFT JOIN NationCustom_CFG NC ON E.Nation_code = NC.code
   LEFT JOIN ( COSTPRICE + CSTPRCSUB + PRCITMLST + PrcItmDtl 4단 집계
               → PRC_PRICE(납부)/RECEIPT(수납)/OSD_PRC(미수)/OP_PRC(수술금액) ) TMP2 ON A.CUST_NUM=TMP2.PRC_CUSNUM
WHERE Exam_Date BETWEEN @from AND @to
  AND (My_Counselor=@counselor OR @counselor='-1')
  AND (My_Doctor=@doctor       OR @doctor='-1')
  AND (My_Optometrist=@optom   OR @optom='-1')
  /* radLast: AND Exam_Date IN (SELECT MAX(Exam_Date) ...) */
  /* CkExcept: AND Cust_num NOT IN (SELECT Cust_num FROM Cataract_Exam WHERE Impossible='Y' OR Stop_YN='Y') */
ORDER BY Exam_Date
```

**radODOS = 좌/우안 각 1행** 분기: 내부 파생테이블에서 `'OD', OPERATIONR, OPERATIONR_DATE, OP_ESTIMATE_R, RIGHT01..15` / `'OS', OPERATIONL, ..., LEFT01..15` 를 `UNION ALL` 로 펼침 (`WHERE OPERATIONR<>''` / `OPERATIONL<>''`). 정렬 `ORDER BY CUST_NUM, EXAM_DATE`.

> 출처: `SoftCrm.decompiled.cs` `FrmCataract_ExamList`(745768~749152). 원본은 **문자열결합 + string.Format**(바인딩 아님) → 이식 시 파라미터 바인딩 필수.

## 3. `Cataract_Exam` 스키마 (실DB · 87컬럼)

| 분류 | 컬럼 |
|------|------|
| 키 | `CUST_NUM`(char13) + **`SEQ`(int)** ← 고객당 복수 검사행 |
| 검사 | `EXAM_DATE`(char10), `EXAM_MEMO`(nvarchar2000), `Stop_YN`, `Cancel_CD`, `Cancel_reason` |
| **검사정보** | `RIGHT01~30` / `LEFT01~30` (각 nvarchar20) — **그리드엔 1~15만 노출** |
| 수술권고 | `OPERATIONR`/`OPERATIONL`(nvarchar40), `Impossible` |
| **견적(좌/우 분리)** | `OP_ESTIMATE_R`/`_L`(nvarchar30, 텍스트), `OP_UnitPrice_R`/`_L`(**money**), `Discount_Code_R`/`_L`, `Estimate_Memo_R`/`_L`(nvarchar500), `Estimate_emp`/`_Date`/`_Gubun` |
| 기타 | `ExamPercent`(영업가율), `Insurance`(varchar100), `InflowCod`(유입), **`FocalDistance`**(초점거리 nvarchar10), `ReasonNum`, `InspcNum` |

> `RIGHT01/LEFT01` = 좌/우 굴절검사값(지표정의 §1.4 백내장 눈 판정 기준과 동일). `RIGHT16~30`은 스키마엔 있으나 그리드/리포트 미사용.

## 4. ★ 시력교정(검사자 리스트)과의 핵심 차이 — 구현 시 주의

| 항목 | 시력교정 (검사자 리스트) | **백내장 (본 화면)** |
|------|------------------------|---------------------|
| 검사 소스 | `EXAM` (고객당 **1행 스냅샷**) | `Cataract_Exam` (`CUST_NUM`+**`SEQ`**, 복수행 가능) |
| 수술 소스 | `OPERATIONDATA` | `Cataract_Operationdata` |
| 수술불가 | `OP_POSSIBLE_YN` | **`Impossible`** |
| 취소사유메모 | `CANCEL_REASON` | **`Cancel_reason`** |
| 검사정보 | RIGHT/LEFT **01~30** (30쌍) | RIGHT/LEFT **01~15** (15쌍, 그리드) |
| 견적가 | `OP_ESTIMATE` (단일) | **좌/우 분리** `OP_ESTIMATE_R`/`_L` + `OP_UnitPrice_R`/`_L`(money) |
| 집도의 | (placeholder) | **좌/우 분리** `OperationR_Doc`/`OperationL_Doc` → EMPLOYEE |
| 다초점 단서 | 없음 | **`FocalDistance`**(초점거리, 단 그리드 미노출) |
| 내원동기·국적·실손·유입 | 검사자리스트 §6 placeholder(미확정) | **본 화면엔 실제 조인 존재**(MOTIVE_NEW02·NationCustom_CFG·EtcCfg) → 시력교정 placeholder 해소 참고 가능 |
| 좌/우안 뷰 | 없음 | **radODOS** 눈 단위 행 분리 지원 |

> ⚠️ **biometry 전용 컬럼(IOL 도수/모델, 안축장 AL, 각막곡률 K, 전방깊이 ACD, 목표굴절) 없음.** 백내장 검사 리스트는 "임상 biometry 리포트"가 아니라 **검사·수술·매출·동기 관리 리포트**다. AL/K/ACD/IOL을 운영상 `RIGHT/LEFT01~15` 자유슬롯에 넣었을 수는 있으나 라벨에 의미가 명시돼 있지 않다(범용 슬롯). 필요하면 검사입력 화면의 항목 매핑표 또는 EyeChartPro 장비연동을 별도 확인.

## 5. 함정 (Gotchas)

- **복수행(SEQ)**: `Cataract_Exam`은 고객당 여러 검사행(`SEQ`) 가능. 시력교정 EXAM처럼 "고객=1행" 가정 금지. `radLast`(최근검사만) 옵션이 `MAX(EXAM_DATE)` 필터로 이를 처리.
- **좌/우안 이원화**: 견적·수술방법·수술일·집도의가 R/L 컬럼으로 갈림. "건수"는 뷰(고객/눈)에 따라 의미가 다름 — 지표정의 §1.4 백내장은 **눈 단위**(RIGHT01·LEFT01 채움) 집계임을 상기.
- **money 타입**: `OP_UnitPrice_R/L`은 money. 수납금액(`RECEIPT`)은 별도 `COSTPRICE` 4단 집계라 단순치 않음(원데이 합산 함정, [tables/COSTPRICE.md](tables/COSTPRICE.md)).
- **검사정보 라벨 부재**: RIGHT/LEFT 인덱스에 고정 임상 라벨 없음(병원/장비 설정 의존). 시력교정과 동일 주의.
- **원본 SQL Injection**: 문자열결합 + string.Format. 이식 시 파라미터 바인딩 필수.
- **JUMIN 복호화**: 보안DB면 `@SEC_JUMIN` 분기 — [tables/CUSTOM.md](tables/CUSTOM.md).

## 6. 다음 단계 (구현 시)

검사자 리스트와 동일 패턴으로 `/api/cataract-exam-list` 구현 가능:
- 소스만 `Cataract_Exam`+`Cataract_Operationdata`로 교체, `Impossible`/`Cancel_reason`/15쌍 반영.
- 내원동기·국적·실손·유입은 본 화면 조인이 확정돼 있어 **placeholder 없이 실데이터** 가능.
- 눈 단위(radODOS) vs 고객 단위(radCustom) 뷰 토글 여부 결정 필요.
