# 검사자 리스트 컬럼 정의서

> SoftCRM **검사자 리스트** 계열 엑셀 출력(75/78컬럼) ↔ java_dashboard API 매핑 문서  
> DB 기준: MSSQL(SOFTCRM), READ-ONLY, 대용량 테이블 `WITH(NOLOCK)` + 날짜 범위 필수  
> 최종 갱신: 2026-06-11

## 요약

| 항목 | 내용 |
|---|---|
| 화면명 | 검사자 리스트 (상담사별) |
| 원본 계열 | `FrmCounselExamList` 구 25컬럼 + 확장 75/78컬럼 SQL |
| 주 데이터 | `EXAM` + `CUSTOM` |
| 기간 기준 | `EXAM.EXAM_DATE` |
| 현재 API | `GET /api/exam-list?from=YYYY-MM-DD&to=YYYY-MM-DD` |
| 현재 구현 | 시력교정 `EXAM` 분기 중심. 백내장 본분기 `UNION` 미구현 |
| 프론트 표시 | 검사정보 측정값 제외, 비측정 47필드 표시 |

> **주의:** 이 화면의 “검사자”는 사람 배정 현황이 아님.  
> 고객별 검사자료 + 예약 + 상담 + 수술 관련 마스터 리스트에 가까움. `uASSIGN1` 검사배정현황과 혼동 금지.

## 출력 기준

java_dashboard 검사자 리스트 API 기준:

- 기간 내 `EXAM.EXAM_DATE` 행 조회
- 상담사/상담의 제한 없이 전체 조회
- 진료구분/예약 부가정보는 최신 검사예약 `RESERVE_FLAG='M'`, `RESERVE_STATE<>'C'` 기준
- 검사분류는 별도 파생값
  - 같은 검사일에 렌즈센터 예약 `D`만 있고 검사예약 `M`이 없으면 `드림렌즈`
  - 그 외 `시력교정`
- 테스트 데이터 제외
  - 고객명에 `테스트`, `test`, `TEST` 포함 시 제외
  - 검사메모/예약번호의 `test` 문구는 제외 기준으로 사용하지 않음
  - `장용테스트`, `모노비전TEST` 같은 정상 임상 메모 오탐 방지 목적
- 백내장 검사 그림자 행 제외
  - 같은 고객/검사일에 `Cataract_Exam` 본행 존재
  - 일반 `EXAM.RIGHT01~RIGHT30/LEFT01~LEFT30` 측정값 전부 공백
  - 위 조건이면 시력교정 검사자 리스트에서 제외
- 모든 응답 필드는 문자열 보장
  - 프론트 Zod 스키마가 `z.string()` 기준이라 `null` 금지

## 데이터 소스

### 원본 기본 구조

```sql
FROM EXAM A
  INNER JOIN CUSTOM B        ON A.CUST_NUM = B.CUST_NUM
  LEFT  JOIN OPERATIONDATA C ON A.CUST_NUM = C.CUST_NUM
  LEFT  JOIN EMPLOYEE D      ON B.MY_COUNSELOR   = D.EMP_NUM
  LEFT  JOIN EMPLOYEE E      ON B.MY_DOCTOR      = E.EMP_NUM
WHERE A.EXAM_DATE BETWEEN @start AND @end
  AND (B.MY_DOCTOR = @doc OR @doc = '-1')
  AND B.MY_COUNSELOR = @userId
```

원본은 상담사 본인 데이터 중심. 대시보드는 전체 현황 목적이라 상담사/상담의 조건 제거함.

### 확장 SQL 추가 조인

- 검사예약: 고객별 최신 `RESERVATION.RESERVE_NUM`
  - `RESERVE_FLAG='M'`
  - `RESERVE_STATE<>'C'`
- 수술예약: 고객별 최신 `RESERVATION.RESERVE_NUM`
  - `RESERVE_FLAG='O'`
  - `RESERVE_STATE<>'C'`
- 진료구분: `MEDICAL_SUB_CFG.SUB_NAME`
  - `RESERVE_FLAG + RESERVE_JINRYO` 매핑
- 수납금액: `COSTPRICE`/`CSTPRCSUB`/`PRCITMLST`/`PrcItmDtl` 집계 `RECEIPT`
- 내원동기: `MOTIVE_NEW02` + `MOTIVE_NEW01`
- 보험사: `EtcCfg`
- 국적: `NationCustom_CFG`

### 백내장 분기

원본 계열은 수술구분에 따라 시력교정/백내장 `UNION` 가능.

| 시력교정 | 백내장 |
|---|---|
| `EXAM` | `Cataract_Exam` |
| `OPERATIONDATA` | `Cataract_Operationdata` |
| `OP_POSSIBLE_YN` | `Impossible` |

현재 API는 시력교정 `EXAM` 중심. 단, 백내장 본행이 있는 일반 `EXAM` 빈 그림자 행은 제외함.

## 검사정보1~30

엑셀 `검사정보N` 구조:

- `EXAM.RIGHT01/LEFT01`부터 `RIGHT15/LEFT15`까지 사용
- 우/좌 15쌍을 엑셀 30칸으로 펼친 구조
- `RIGHT16~30/LEFT16~30`은 DB에 존재하나 원본 엑셀 30칸과 1:1 대응 아님

| 검사정보 | 우안 | 좌안 | 샘플 의미 |
|---:|---|---|---|
| 1 | `RIGHT01` | `LEFT01` | 구면(Sph) |
| 2 | `RIGHT02` | `LEFT02` | 난시(Cyl) |
| 3 | `RIGHT03` | `LEFT03` | 축(Axis) |
| 4 | `RIGHT04` | `LEFT04` | 시력 |
| 5~7 | `RIGHT05~07` | `LEFT05~07` | 시력/굴절 계열 추정 |
| 8 | `RIGHT08` | `LEFT08` | 안압(IOP) |
| 9 | `RIGHT09` | `LEFT09` | 각막두께(CCT) |
| 10~12 | `RIGHT10~12` | `LEFT10~12` | 검사 장비/폼 설정 의존 |
| 13 | `RIGHT13` | `LEFT13` | 전방깊이(ACD) 추정 |
| 14 | `RIGHT14` | `LEFT14` | 각막직경(WTW) 추정 |
| 15 | `RIGHT15` | `LEFT15` | 검사 장비/폼 설정 의존 |

> RIGHT/LEFT 인덱스 고정 라벨 없음.  
> 위 의미는 샘플 기반 추정. 정확한 항목명 필요 시 검사입력 화면/장비 매핑표 별도 확인 필요.

## 주의사항

- 원본 SQL은 문자열 결합 방식. 이식 시 파라미터 바인딩 필수
- 원본 `FrmCounselExamList` 구버전은 25컬럼. 운영 엑셀 기준은 확장 75/78컬럼 SQL
- 원본은 상담사 본인 데이터 필터 존재. java_dashboard는 전체 조회 기준
- 백내장 리스트까지 같은 API에서 보여야 하면 `Cataract_Exam`/`Cataract_Operationdata` `UNION` 설계 필요
- 루트 엑셀은 75열, 원본 정의는 78열 계열. 엑셀 export 옵션에 따라 최근내원일/보험사/국적 누락 가능
- `CUSTOM.MY_OPTOMETRIST`는 고객 담당 검안사 기준. 검사일 실제 수행자와 다를 수 있음
- 주민번호 복호화는 보안/성능 리스크 큼. API 출력 제외 유지 권장
