## 1. 검사건수

검사는 **시력교정 · 백내장 · 드림렌즈** 3종으로 구분한다. **각 카테고리는 소스·집계단위가 서로 다르다.**

### 1.1 개요

| 카테고리 | 소스 테이블 | 분류 조건 | 집계 단위 | 2026-04 / 05 |
|---|---|---|---|---|
| **시력교정검사** | `EXAM` (검사자 리스트) | `EXAM_DATE` 기준, **드림렌즈 분류 제외** (측정값 무관) | **행/사람** | 809 / 900 |
| **백내장검사** | `Cataract_Exam` | 실제 검사 기록, 취소/중단 제외 | **눈** (좌/우 각각) | 214 / 256 |
| **드림렌즈검사** | `EXAM` (검사자 리스트) | 같은 검사일 D/L 계열 `D`만 있고 검사 `M`이 없는 행 | **행/사람** | 재검증 필요 |

> ⚠️ **단위가 달라 단순 합산은 의미가 다르다** (시력교정·드림렌즈=사람, 백내장=눈). 통합 "총 검사건수"가 필요하면 단위를 통일하는 별도 합의가 필요하다.

### 1.2 공통 규칙

**검사결과 테이블 기반** (시력교정=`EXAM`, 백내장=`Cataract_Exam`)
- 날짜 기준: `EXAM_DATE` (char(10), `YYYY-MM-DD`) — 실제 검사일
- **백내장은 "측정값 존재"(`RIGHT01`/`LEFT01`)로 실시 판정**. **시력교정은 측정값 무관** — 병원 검사자 리스트 행수 기준이라 측정값 미입력 검사도 포함.
- 더 엄격한 "완료" 기준이 필요하면 `STOP_YN='Y'`(중단)·`CANCEL_CD`(취소)를 추가 제외.

**RESERVATION 기반** (드림렌즈)
- 소스: `RESERVATION WITH(NOLOCK)`, 날짜 범위 필수
- 상태: `RESERVE_STATE IN ('I','H')` — 예약(Y)·취소(C) 제외. 이 병원은 'I'에서 멈추므로 사실상 **"내원(체크인)"**.
- 날짜 기준: `RESERVE_DATE`
- 정기검진(FU)은 제외.

### 1.3 시력교정검사

**정의**: 해당 기간에 시력교정 검사를 실제 받은 **사람 수**. 소스는 검사결과 테이블 `EXAM` (병원 "검사자 리스트(상담사별)" 화면과 동일 소스). **EXAM에 섞인 드림렌즈는 제외.**

| 항목 | 값 |
|------|-----|
| 소스 | **`EXAM`** (시력교정 검사결과, PK=`CUST_NUM`) |
| 날짜 기준 | `EXAM_DATE` (char(10), `YYYY-MM-DD`) |
| 측정값 필터 | **없음** — 병원 검사자 리스트 행수 기준(측정값 미입력 검사도 포함). 엄격 "실시"분만 필요하면 `RIGHT01`/`LEFT01` 존재 추가 |
| 드림렌즈 제외 | 같은 날 D/L 계열(`RESERVE_FLAG='D'`, `RESERVE_JINRYO IN ('1','5','7')`) 예약만 있고 시력교정(`'M'`) 예약 없는 EXAM 배제 |
| 집계 단위 | **사람** = `COUNT(DISTINCT CUST_NUM)` (EXAM은 고객당 1행이라 행수와 동일) |

> **소스가 RESERVATION이 아니라 `EXAM`인 이유**: 병원 공식 "검사자 리스트(상담사별)" 화면(`FrmCounselExamList`)이 시력교정을 `EXAM`+`EXAM_DATE`로 집계한다. "실제 검사 실시" 기준이며 백내장(`Cataract_Exam`)과 동일 구조다. RESERVATION(flag M) 예약기반 집계는 검사 실시와 불일치(과거 정의 843/805 → 본 정의로 교체).
> **드림렌즈 혼입 주의**: `EXAM`에는 검사종류 구분 컬럼이 없어 드림렌즈(CRT/각막굴절교정) 검사가 섞여 있다. 같은 날 D/L 계열 예약 기준으로 배제한다. H/L(`D-8` 등)은 드림렌즈로 보지 않음.

```sql
SELECT SUBSTRING(e.EXAM_DATE,1,7) AS ym,
       COUNT(DISTINCT e.CUST_NUM) AS visionCorrection
FROM EXAM e WITH(NOLOCK)
WHERE e.EXAM_DATE >= :from AND e.EXAM_DATE <= :to
  AND NOT (                                    -- 드림렌즈 배제
        EXISTS (SELECT 1 FROM RESERVATION rd WITH(NOLOCK)
                WHERE rd.CUST_NUM = e.CUST_NUM AND rd.RESERVE_DATE = e.EXAM_DATE
                  AND rd.RESERVE_STATE IN ('I','H') AND rd.RESERVE_FLAG = 'D')
    AND NOT EXISTS (SELECT 1 FROM RESERVATION rm WITH(NOLOCK)
                WHERE rm.CUST_NUM = e.CUST_NUM AND rm.RESERVE_DATE = e.EXAM_DATE
                  AND rm.RESERVE_STATE IN ('I','H') AND rm.RESERVE_FLAG = 'M')
  )
GROUP BY SUBSTRING(e.EXAM_DATE,1,7)
ORDER BY ym
```

- 보조(엄격 "실시" 기준): 위 조건 + 측정값 존재(`RIGHT01`/`LEFT01`) → 측정 완료분만 02=1585 / 03=850 / 04=799 (데스크는 미입력도 세므로 기본 정의는 측정값 무관).
- **검증 (병원 수기 대조)**: 02=1601(병원 1604) / 03=855(853) / 04=810(807) / 05=901 — 전월 **±0.4%**. 04월은 병원 엑셀 826행 = EXAM 4월 전체(827, ±1)와 1:1 대조로 확정. 드림렌즈 미제외 시 02=1612/03=874/04=827.

### 1.4 백내장검사

**정의**: 해당 기간에 실제 실시된 백내장 검사를 **검사한 눈(좌/우) 단위**로 집계. 양안 검사 = 2건, 단안 = 1건.

| 항목 | 값 |
|------|-----|
| 소스 | **`Cataract_Exam`** (실제 백내장 검사 상세, `CUST_NUM`+`SEQ`) |
| 날짜 기준 | `EXAM_DATE` (char(10), `YYYY-MM-DD`) |
| 제외 | `Stop_YN='Y'`(중단), `Cancel_CD` 채워짐(취소) |
| 눈 판정 | 우안 = `RIGHT01` 채워짐, 좌안 = `LEFT01` 채워짐 (굴절검사값 존재) |
| 집계 단위 | **눈 (좌/우 각각 1건)** = `SUM(RIGHT01 채움) + SUM(LEFT01 채움)` |

> **소스가 RESERVATION이 아니라 `Cataract_Exam`인 이유**: 좌/우안 구분은 예약 테이블에 없고, 실제 검사값(좌/우 굴절)은 이 테이블에만 존재한다. 또 "실제 검사 실시 기록" 기준과도 일치한다.
> **실측 특성**: 단안(only_right/only_left) 검사는 0건. 전부 양안 또는 측정값 없음(월 1~3건). 따라서 눈 단위 ≈ 유효검사 × 2.

```sql
SELECT SUBSTRING(c.EXAM_DATE,1,7) AS ym,
       SUM(CASE WHEN c.RIGHT01 IS NOT NULL AND c.RIGHT01 <> '' THEN 1 ELSE 0 END)
     + SUM(CASE WHEN c.LEFT01  IS NOT NULL AND c.LEFT01  <> '' THEN 1 ELSE 0 END) AS cataractExamEyes
FROM Cataract_Exam c WITH(NOLOCK)
WHERE c.EXAM_DATE >= :from AND c.EXAM_DATE <= :to
  AND (c.Stop_YN IS NULL OR c.Stop_YN <> 'Y')
  AND (c.Cancel_CD IS NULL OR LTRIM(RTRIM(c.Cancel_CD)) = '')
GROUP BY SUBSTRING(c.EXAM_DATE,1,7)
ORDER BY ym
```

- 보조: 검사인원 = `COUNT(DISTINCT c.CUST_NUM)`, 검사세션 = `COUNT(*)`.
- **검증**: 03=254 / **04=214** / **05=256** 눈. (검사세션: 128 / 110 / 131)
- ✅ **재검사 처리**: 동일고객 월 복수검사(복수 `SEQ`)는 월 ≤2눈으로 사실상 없음(04·05월 0건) → **별도 제외 안 함**. 각 세션이 실제 검사 실시 기록이므로 그대로 카운트.

### 1.5 드림렌즈검사

**정의**: 해당 기간 드림렌즈(D/L) 계열 검사 건수. **H/L, MiSight, FU, 소아검진 등 렌즈센터 비-D/L 세부구분 제외.**

| 항목 | 값 |
|------|-----|
| 소스 | `RESERVATION` |
| 진료유형 | `RESERVE_FLAG = 'D'` 중 D/L 계열 |
| 상태 | `RESERVE_STATE IN ('I','H')` |
| 세부유형 포함 | `RESERVE_JINRYO IN ('1','5','7')` = D/L, D/L_재구매, D/L_신규 |
| 집계 단위 | **`COUNT(DISTINCT CUST_NUM)` (사람단위)** — 시력교정과 통일 |

> 렌즈센터(`D`)는 H/L, MiSight, 소아검진까지 섞이므로 드림렌즈 화면에서는 D/L 계열만 포함함. 2026-02의 `D-8=H/L_신규` 3건은 드림렌즈가 아니라 시력교정 쪽으로 분류.

```sql
SELECT YEAR(r.RESERVE_DATE) AS yr, MONTH(r.RESERVE_DATE) AS mo,
       COUNT(DISTINCT r.CUST_NUM) AS dreamlensPersons
FROM RESERVATION r WITH(NOLOCK)
WHERE r.RESERVE_DATE >= :from AND r.RESERVE_DATE <= :to
  AND r.RESERVE_FLAG = 'D'
  AND r.RESERVE_STATE IN ('I','H')
  AND ISNULL(r.RESERVE_JINRYO, '') NOT IN ('2','4','11')   -- FU 제외
GROUP BY YEAR(r.RESERVE_DATE), MONTH(r.RESERVE_DATE)
ORDER BY yr, mo
```

- **검증**: 03=45 / 04=55 / 05=60 명. (건수 기준과 0~3건 차이로 거의 동일)

### 1.6 함정 (Gotchas)

- **카테고리별 소스·단위 상이**: 시력교정=`EXAM`/사람, 백내장=`Cataract_Exam`/눈, 드림렌즈=`RESERVATION`/사람. 코드·차트에 소스/단위 라벨 명시 필수.
- **시력교정 소스 주의**: 예약(`RESERVATION` flag M)이 아니라 **검사결과(`EXAM`)** 기준. 예약기반은 병원 공식 수치와 어긋난다(예약기반 843/805 vs 검사결과 855/810, 병원 853/807).
- **EXAM에 드림렌즈 혼입**: `EXAM`엔 검사종류 구분 컬럼이 없어 드림렌즈(CRT/각막굴절교정) 검사가 월 ~2%(12-24명) 섞인다. 같은 날 렌즈센터(`D`) 예약 기준으로 반드시 배제(미배제 874/827 → 배제 855/810).
- **측정값 필터는 시력교정 기본 정의에서 미적용**: 병원 데스크가 측정값 미입력 검사도 리스트에 세므로 행수 기준으로 맞춘다. 측정값 존재(`RIGHT01`/`LEFT01`) 조건은 엄격 "실시" 보조지표로만(측정분 1585/850/799). 백내장은 측정값으로 실시 판정.
- **EXAM은 스냅샷(고객당 1행) → 과거 월 수치가 사후 감소**: 고객이 나중에 재검사하면 `EXAM_DATE`가 덮어써져 원래 달에서 빠진다. 실측 월 -3~-5명(~0.3%) 수준. 과거 월 수치는 조회 시점에 따라 미세하게 달라질 수 있음.
- **측정값 미입력 검사(월 ~12명)**: `EXAM_DATE`는 해당월인데 `RIGHT01`/`LEFT01`이 빈 검사. 병원 데스크와 본 정의(측정값 무관)는 포함, 엄격 "실시" 보조지표에선 제외.
- **백내장 단위 혼동 금지**: "눈" 단위라 인원의 약 2배. 시력교정(사람)과 직접 비교 불가.
- **드림렌즈 FU 제외 영향 큼**: FU가 65% → 정의 변경 시 수치 급변. 비교 시점 정렬 주의.
- **드림렌즈 상태값 `I` 단독**: `RESERVATION` `IN('I','H')`는 사실상 "내원(체크인)"이지 검사 완료 보장이 아님.
- **병원 수기집계와 ±오차 정상**: 병원 자체 수치(시력교정 03=853/04=807 등)는 수기/엑셀 집계라 시스템 정의와 1~2% 차이는 자연스럽다.

### 1.7 검증 이력 (2026년)

| 카테고리 | 단위 | 2026-02 | 2026-03 | 2026-04 | 2026-05 |
|---|---|--:|--:|--:|--:|
| 시력교정검사 | 사람 (EXAM, 드림렌즈 분류 제외) | 1,600 | 854 | 809 | 900 |
| 백내장검사 | 눈 (좌/우) | 200 | 254 | 214 | 256 |
| 드림렌즈검사 | 사람 (EXAM, 검사자 리스트 분류) | 11 | 19 | 17 | 23 |

> 1·2월은 겨울방학 시즌이라 시력교정이 3~5월의 약 2배.

**시력교정 ↔ 병원 자체 수기집계 대조**

| 구성 요소 | 2026-02 | 2026-03 | 2026-04 |
|---|--:|--:|--:|
| EXAM 해당월 전체 | 1,612 | 874 | 827 |
| − 드림렌즈 배제 | −11 | −19 | −17 |
| **= 정의값** (드림렌즈 제외, 측정값 무관) | **1,601** | **855** | **810** |
| **병원 수기** | **1,604** | **853** | **807** |
| 잔차 | +3 | −2 | −3 |

> 잔차 ±3(≤0.4%)는 **EXAM 스냅샷 덮어쓰기**(과거월일수록 미세 과소, 월 +3~5)와 수기집계 노이즈로 설명된다. 04월은 병원 엑셀(826행) = EXAM 4월 전체(827, ±1)와 1:1 대조로 확정. 참고로 측정값까지 요구하는 엄격 "실시" 기준은 1585/850/799(데스크 대비 −1%).

### 1.8 코드 반영 상태

- [ExaminationStatsRepository.java](../../backend/src/main/java/com/bviit/analytics/repository/stats/ExaminationStatsRepository.java):
  - 시력교정: `EXAM` + 드림렌즈 배제, **측정값 필터 제거 완료** (`findVisionCorrectionMonthly`) ✅
  - 드림렌즈: 검사자 리스트와 같은 `EXAM` 기준 (`findDreamlensMonthly`) ✅
  - 백내장/외래: `/stats/examination` 화면과 API에서 제거
- API 응답(`ExaminationMonthlyItem`)은 `visionCorrection`, `dreamlens`, `examTotal`, `total`만 사용.
- 실DB 검증: 02~05월 시력교정 **1600/854/809/900**, 드림렌즈 **11/19/17/23**. 두 항목 합계는 검사자 리스트 월별 건수와 일치.
- `/stats/examination`은 기본 JSON 캐시 사용. 캐시 파일이 없으면 DB 쿼리 후 `.cache/stats/examination-monthly-v5-*.json`에 저장.
- 2024~2025년 `시력교정검사`(`visionCorrection`)는 레거시 출력 확정값을 사용. DB 재집계로 업데이트하지 않음.
- 화면 `전체`(`examTotal`)는 항상 `시력교정검사 + 드림렌즈검사`.
