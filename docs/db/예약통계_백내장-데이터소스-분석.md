## 2. 백내장 분류 코드 사전

### 2.1 컨택센터 콜 — `CtiRptLst`(+`CtiClg`)
코드 = `CtiGbnCod / CtiCtgCod / CtiDtlCod1 / CtiDtlCod2 (/ CtiDtlCod3)`

| 분류 | 코드 |
|---|---|
| 총 신규검사문의 | `A/H/10/28`(★신환) · `S/9/21/21`(신규문의, Cod3<>'28') · `S/9/21/9`(★신환) · `S/9/21/32`(☆발신신규) · `A/H/10/32`(★발신신환) · `S/9/21/41`(★발신신환) · `M/H/7/5`(예약수정 신환) |
| 신규 재문의 | `S/9/21/21` + `Cod3='28'`(신환재상담), 수신만(`ClgIOF='I'`) |
| ★신환(예약) | `A/H/10/28` · `A/H/10/32` · `S/9/21/41` · `A/H/10/28/3`(대리인) · `A/H/10/31`(추가예약) · `M/H/7/5` + **메모 가산** |
| 재문의 비율 | 재문의 ÷ 신규 (파생) |

- **메모 가산**: `CtiRptMmo`에 `+1~+5` → 건수 +N(2~6), `/1~/5` → −N. 즉 **신환 CNT가 1이 아닐 수 있음**(추가예약/취소를 메모로 표기).
- **발/수신**: `CtiClg.ClgIOF` = `I`(수신) / `O`(발신). 시력교정 RSS는 `ClgIOF='I'`(수신만), 백내장 콜마감은 이 필터를 **주석 처리**해 발신 포함.

### 2.2 카톡 — `HappyTalk_Counsel_List`(+Mapping/Category01·02)
- **대분류(Category01)**: `1 수술전` · `2 수술후외래` · `3 일반외래` · `4 불량` · `5 백내장검사` · `6 백내장외래` · `7 백내장수술`
- **노안**: `Category02 노안(Pkey=48)` — 부모는 **`백내장외래`(C01=6)**. (검사가 아닌 외래 하위, 노안 카테고리는 이것 하나뿐)
- 검사예약 = 백내장검사 × `★신환(1~4)` (★신환N이면 CNT=N), 신규문의 = `☆신규문의`, 취소 = `예약취소`

## 4. 상세 근거 (운영DB 실측)

### 4.1 EICN 인입콜 큐 — ✅ 해결(상담원 그룹 기준, 2026-06-25)
**큐(hunt_number)가 아니라 응대 상담원 그룹(`group_code`)으로 분리된다.** 인입 큐는 시력교정/백내장이 공유한다.

2026-06-23 `OPENQUERY(EICN_MySQL, stat_user_inbound_bseye)` 그룹별:

| group_code | tree | 인원 | 인입/응대 | 정체 |
|---|---|---|---|---|
| `0015` | `0004_0009_0015` | 7명 | 195/194 | ✅ 시력교정 팀 |
| **`0016`** | `0004_0009_0016` | 2명 (user249·user276) | **29/29** | ✅ **백내장 팀** (사용자 확정 6/23=29와 정확 일치) |
| `0017` | `0004_0010_0017` | 1명 | 1/1 | 기타 |

- 백내장 = `group_code='0016'`, `dcontext='hunt_context'`, `SUM(in_total)`=인입콜·`SUM(in_success)`=응대콜. 라이브 적용됨(CH_01, `cataract-daily-counts.sql`).
- hunt_number 후보(`00033331103`=109·`00022221103`=13)는 시력교정과 공유라 6/23 29와 불일치 → 그룹 방식이 정답. 시력교정 RSS(hunt 1121)는 hunt에 섞인 백내장그룹 콜 5건(6/23)도 같이 셈 → 그룹 방식이 더 정확.
- PDF 1~22일 대조: 응대콜 대부분 정확. 인입콜은 ±1~2 차이(PDF는 내부전환콜 `dcontext='inbound_inner'` 포함 + 일부 수기보정). 6/17만 큰 차이(PDF 30 vs EICN 19) = user249가 그날 EICN 기록 0(부재) → **PDF 인입콜은 순수 EICN 아님**. 따라서 1~22일은 PDF값 유지, 23일~ EICN 라이브.

### 4.2 DB_CUSTOM (아웃바운드 DB) — ✅ TM_EMP로 백내장 TM 분리 가능 (2026-06-24 정정)
- `Gubun` = **마케팅 유입 채널**: H_전화상담관리 · H_이벤트상담신청 · M_모두닥이벤트 · M_강남언니이벤트 · M_바비톡 · 여성시대 · **B2B_군_…**
- `TM_Gubun` = **TM 단계코드**: 1000(예약)·2000(유효)·3000~6000 → `예약=1000` · `유효DB=1000+2000` · `총DB=전체`
- **핵심**: 질환 구분자는 없지만 **담당자(TM_EMP)로 백내장 TM팀이 분리됨**. 시력교정 RSS의 CH_04는 `NOT TM_EMP IN('KC0307','BV1119','BV1207','BV0067')`로 이 4명을 **제외** → 그 4명이 바로 **백내장 TM팀**. (시력교정 = DB_CUSTOM 전체 − 백내장4명)
- **백내장 TM = `DB_CUSTOM WHERE TM_EMP IN(4명) AND ISNULL(Gubun,'') NOT LIKE 'B2B%'`** (assign_date 기준). B2B 군 리드는 백내장 TM 퍼널이 아니므로 제외(미제외 시 월요일 등에서 PDF 대비 과집계).
- **검증**: 2026-06 18영업일 전부 PDF(노안백내장 RSS 수기 엑셀) 총DB/유효DB/예약 **정확 일치**. 예) 6/23 6/3/2 · 6/22 3/0/0 · 6/15 7/5/3.

### 4.3 발신콜 (outbound call) — ✅ 측정 가능
2026-06-01~23 백내장 콜(CtiRptLst, `CtiCtgCod='H' Cod1='10'/'7'` 또는 `'9'/'21'`):

| ClgIOF | 건수 |
|---|---|
| I (수신) | 109 |
| **O (발신)** | **49** |
| (CtiClg 없음·수기) | 32 |

→ 백내장도 상담사가 발신을 돌림. **발신콜수·발신 신환·발신 예약**을 `ClgIOF='O'` + 발신코드(`S/9/21/32`·`A/H/10/32`·`S/9/21/41`)로 집계 가능. 단 시력교정식 **총DB/유효DB 퍼널과는 형태가 다름**.

### 4.4 노안 구분 — ❌ CRM 독립차원 부재
- RESERVATION FLAG='H': 노안 구분자 없음 (`RESERVE_JINRYO`는 진료실, 공란 최다)
- DB_CUSTOM: 노안 없음
- HappyTalk: `백내장외래 > 노안`(C02 Pkey48) **한 곳뿐**
- → 총예약 백내장/노안 분리는 근거 없음(**합계만**), "노안 검사예약"도 외래 카테고리라 재정의 필요.

### 4.5 온라인 — ✅ 가능하나 ≈0
2026-06 RESERVATION FLAG='H'(I/H/Y) 경로: CRM 719 · CTI 99 · **ONLINE 1 · NAVER 0**. → 백내장은 사실상 CRM/CTI 유입, 온라인 컬럼은 채워도 ~0.

### 6.1 캘리브레이션 (2026-06-24, 라이브 적용)
라이브가 PDF보다 부풀던 두 지표를 PDF 정의에 맞게 교정 (PDF 1~22일 대조):
- **내원 = `RESERVATION`(FLAG='H', 내원 STATE `I`접수/`H`퇴원) 중 ①신규 검사 슬롯(진료실 일반검사`1`·OP전검사`2`) 또는 ②같은 날 `Cataract_Exam`(중단 Stop_YN<>'Y' 제외) 기록 보유분** (2026-06-25 교정). 레거시 roster(RESERVATION I/H, §1 Q1)+PDF 내원과 6/15~24 **전부 일치**(6/16 2·6/19 6·6/20 4·6/22 4·6/23 9·6/24 4).
  - **Cataract_Exam 단독 방식 폐기 이유**: 이전 정의(`Cataract_Exam` ∩ 같은날 I/H 예약)는 RESERVATION I/H의 **부분집합**이라 구조적으로 ≤ 레거시. **일반검사(JINRYO=1)로 내원했으나 Cataract_Exam에 검사기록이 안 생기는 환자**를 놓쳐 6/24 우리 2 vs 레거시 4 발생(누락 2명 = CUST 2232537·2232528, 둘 다 JINRYO=1 STATE H). → STATE 기반으로 교정.
  - **진료실 카탈로그(MEDICAL_SUB_CFG FLAG='H')**: 1=C_일반검사·2=C_OP전검사·3=C_재검사·4=C_당일재검사·13=수술당일·16=진료실. 내원은 **신규검사(1·2)만** 무조건 포함, 그 외(재검사 3·4 등)는 **Cataract_Exam 기록이 있을 때만** 포함(예: 6/22 CUST 2020314 JINRYO=10이나 검사기록 보유 → 포함). 6/20 CUST 2088219(JINRYO=3 재검사·검사기록 없음)는 PDF도 제외 → 4 일치.
  - **협진/타과 자연 배제**: FLAG='H' 예약이 없는 협진 검사는 애초에 안 잡힘(별도 EXISTS 필터 불필요). 더미고객(8888)·테스트 고객명 제외.
  - 저장 스냅샷은 라이브였던 **6/24만 내원 2→4 패치**(6/15~23은 PDF/기존값이 새 공식과 동일).
