# 예약통계_백내장 — 데이터소스 분석

> **목적**: 백내장 예약통계(`reservation-stats-cataract`) 페이지의 채널별 데이터소스가 운영DB로 채워지는지(가능/불가) 정리
> **방법**: 레거시 "백내장 콜마감" 7쿼리 + 시력교정 RSS SQL([ReservationStatsSystemRepository](../../backend/src/main/java/com/bviit/analytics/repository/reservation/ReservationStatsSystemRepository.java)) + 운영DB(MSSQL mssql-prod) 실측
> **조사일**: 2026-06-24
> **핵심 키**: 백내장 = `RESERVATION.RESERVE_FLAG='H'`, 내원/예약 = `RESERVE_STATE in ('I','H','Y')`, 진료실 제외 = `RESERVE_JINRYO<>'16'`

## 2. 백내장 분류 코드 사전

### 2.1 컨택센터 콜 — `CtiRptLst`(+`CtiClg`)
코드 = `CtiGbnCod / CtiCtgCod / CtiDtlCod1 / CtiDtlCod2 (/ CtiDtlCod3)`

| 분류 | 코드 |
|---|---|
| 총 신규검사문의 | `A/H/10/28`(★신환) · `S/9/21/21`(신규문의, Cod3<>'28') · `S/9/21/9`(★신환) · `S/9/21/32`(☆발신신규) · `A/H/10/32`(★발신신환) · `S/9/21/41`(★발신신환) · `M/H/7/5`(예약수정 신환) |
| 신규 재문의 | `S/9/21/21` + `Cod3='28'`(신환재상담), 수신만(`ClgIOF='I'`) |
| ★신환(예약) | `A/H/10/28` · `A/H/10/32` · `S/9/21/41` · `A/H/10/28/3`(대리인) · `A/H/10/31`(추가예약) · `M/H/7/5` + **메모 가산** |
| 재문의 비율 | 재문의 ÷ 신규 (파생) |

- **메모 가산**: `CtiRptMmo`에 `+1~+5` → 건수 +N(2~6), `/1~/5` → −N. 즉 **신환 CNT가 1이 아닐 수 있음**(추가예약/취소를 메모로 표기).
- **발/수신**: `CtiClg.ClgIOF` = `I`(수신) / `O`(발신). 시력교정 RSS는 `ClgIOF='I'`(수신만), 백내장 콜마감은 이 필터를 **주석 처리**해 발신 포함.

### 2.2 카톡 — `HappyTalk_Counsel_List`(+Mapping/Category01·02)
- **대분류(Category01)**: `1 수술전` · `2 수술후외래` · `3 일반외래` · `4 불량` · `5 백내장검사` · `6 백내장외래` · `7 백내장수술`
- **노안**: `Category02 노안(Pkey=48)` — 부모는 **`백내장외래`(C01=6)**. (검사가 아닌 외래 하위, 노안 카테고리는 이것 하나뿐)
- 검사예약 = 백내장검사 × `★신환(1~4)` (★신환N이면 CNT=N), 신규문의 = `☆신규문의`, 취소 = `예약취소`

## 4. 상세 근거 (운영DB 실측)

### 4.1 EICN 인입콜 큐 — 🔶 라벨 미확인
2026-06-23 `OPENQUERY(EICN_MySQL, stat_user_inbound_bseye)` 인입 큐:

| hunt_number | 콜수 | 정체 |
|---|---|---|
| `00000001121` | 102 | ✅ 시력교정 검사 (RSS가 쓰는 번호. 짝: `07040180093`) |
| `00033331103` | 109 | ❓ 라벨 없음 (최다) |
| `00022221103` | 13 | ❓ 라벨 없음 |
| `00000000059` | 1 | 기타 |

- EICN에 **큐 이름표 테이블 없음**(`%hunt%/%queue%/%did%/%route%/%group%` 0건), stat 테이블에도 이름 컬럼 없음(group_code·category 코드만).
- → **데이터만으로 백내장 큐 확정 불가.** 후보 `00033331103`·`00022221103`. **병원에 "백내장 대표번호" 확인되면** 그 hunt_number로 시력교정 CH_01처럼 연결 가능.
- ※ 레거시 백내장 콜마감은 EICN 미사용 — 인입콜 원수는 원래 백내장 리포트에 없던 지표.

### 4.2 DB_CUSTOM (아웃바운드 DB) — ❌ 질환 구분 없음
- `Gubun` = **마케팅 유입 채널**: H_전화상담관리 · H_이벤트상담신청 · M_모두닥이벤트 · M_강남언니이벤트 · M_바비톡 · 여성시대 · B2B_군_…
- `TM_Gubun` = **TM 단계코드**: 1000(예약)·2000(유효)·3000~6000
- → 전부 **라식/라섹(시력교정) 마케팅 리드**. 백내장 리드 없음 → 백내장 TM-DB 분리 불가.
- **시력교정 아웃바운드 = DB_CUSTOM 전체**(질환필터 없이, TM_Gubun으로 단계분류). DB_CUSTOM 자체가 시력교정 전용이라 가능한 것. 백내장에 "전체"를 쓰면 시력교정 리드를 오집계.

### 4.3 발신콜 (outbound call) — ✅ 측정 가능
2026-06-01~23 백내장 콜(CtiRptLst, `CtiCtgCod='H' Cod1='10'/'7'` 또는 `'9'/'21'`):

| ClgIOF | 건수 |
|---|---|
| I (수신) | 109 |
| **O (발신)** | **49** |
| (CtiClg 없음·수기) | 32 |

→ 백내장도 상담사가 발신을 돌림. **발신콜수·발신 신환·발신 예약**을 `ClgIOF='O'` + 발신코드(`S/9/21/32`·`A/H/10/32`·`S/9/21/41`)로 집계 가능. 단 시력교정식 **총DB/유효DB 퍼널과는 형태가 다름**.

### 4.4 노안 구분 — ❌ CRM 독립차원 부재
- RESERVATION FLAG='H': 노안 구분자 없음 (`RESERVE_JINRYO`는 진료실, 공란 최다)
- DB_CUSTOM: 노안 없음
- HappyTalk: `백내장외래 > 노안`(C02 Pkey48) **한 곳뿐**
- → 총예약 백내장/노안 분리는 근거 없음(**합계만**), "노안 검사예약"도 외래 카테고리라 재정의 필요.

### 4.5 온라인 — ✅ 가능하나 ≈0
2026-06 RESERVATION FLAG='H'(I/H/Y) 경로: CRM 719 · CTI 99 · **ONLINE 1 · NAVER 0**. → 백내장은 사실상 CRM/CTI 유입, 온라인 컬럼은 채워도 ~0.

## 6. 권장 — 백내장 페이지 컬럼 재정의

데이터가 받쳐주는 컬럼 위주로:
- **유지**: 컨택센터 콜분류(신규·신환·재문의) · 카톡(검사/외래) · 온라인(≈0) · 취소 · 총예약 합계
- **재정의**: 아웃바운드 `총DB/유효DB/예약` → **`발신콜수 / 발신 신환 / 발신 예약율`**(§4.3)
- **제거/보류**: 백내장/노안 분리(합계만), 노안 검사예약, (미확인)인입콜
