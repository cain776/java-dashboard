## 4. 일치하지 않는 부분 — 원인 및 해결방안

### 🔴 P0. 백내장 예약률 쿼리 단위 미스매치

**위치:** `backend/src/main/java/com/bviit/analytics/repository/stats/ExaminationStatsRepository.java:161-203`

**문제 코드:**
```sql
SELECT base.yr,
       base.mo,
       SUM(base.rightEye) + SUM(base.leftEye) AS examCount,        -- 분모: 눈 단위 ❌
       SUM(CASE WHEN (base.rightEye = 1 OR base.leftEye = 1)
                  AND base.hasSurgeryBooking = 1 THEN 1 ELSE 0 END) -- 분자: 사람 단위
       AS surgeryBookedCount
FROM (...) base
GROUP BY base.yr, base.mo
```

**원인:** 분모(`examCount`)는 **눈 단위(좌+우 양안 검사 시 2)**, 분자(`surgeryBookedCount`)는 **사람 단위**. 단위가 일치하지 않아 예약률이 항상 실제보다 낮게 계산됨.

**해결 옵션:**

**옵션 A (권장)** — 둘 다 **사람 단위**로 통일 (Excel과 일치):
```sql
COUNT(DISTINCT CASE WHEN rightEye = 1 OR leftEye = 1 THEN cust_num END) AS examCount,
SUM(CASE WHEN (rightEye = 1 OR leftEye = 1)
           AND hasSurgeryBooking = 1 THEN 1 ELSE 0 END) AS surgeryBookedCount
```
> `base` 서브쿼리 SELECT 절에 `ce.CUST_NUM AS cust_num` 추가 필요.

**옵션 B** — 둘 다 **눈 단위**로 통일 (양안 예약 시 2건으로 카운트):
```sql
SUM(rightEye) + SUM(leftEye) AS examCount,
SUM(CASE WHEN rightEye = 1 AND hasSurgeryBooking = 1 THEN 1 ELSE 0 END)
+ SUM(CASE WHEN leftEye = 1 AND hasSurgeryBooking = 1 THEN 1 ELSE 0 END) AS surgeryBookedCount
```

화면 라벨이 "예약률(%)"이므로 **옵션 A**가 Excel과 일치합니다.

### 🔴 P1. "검사건수" 메뉴 — 컬럼 의미 매핑 오류

**위치:** `backend/src/main/java/com/bviit/analytics/repository/stats/ProcedureExamStatsRepository.java` + `frontend/src/pages/ProcedureExamPage.tsx`

**문제:**

| 항목 | 값 (2024-01) | 의미 |
|---|---|---|
| API `examCount` | 2,432 | **총검사자** (= Excel C) |
| Excel "일반검사 (AJ)" | 1,007 | 시력교정에서 원데이를 뺀 값 |
| 차이 | +1,425 | 화면 라벨 "일반검사"인데 실제는 총검사자 |

`AG(시력교정 2134) = AH(원데이 1127) + AJ(일반검사 1007)` — Excel은 "일반검사"를 "비-원데이 시력교정"으로 정의.

**해결 옵션:**

- **옵션 A (단순)** — 화면 라벨 정정. "일반검사" → "총검사자" 또는 "전체 검사자".
- **옵션 B (Excel 일치)** — API에 새 필드 추가:
  ```java
  // ProcedureExamMonthlyItem.java
  int generalExamCount;   // = visionExamCount - oneDayExamCount
  ```
  쿼리에서 `visionCorrection - oneDayExamCount` 계산하여 추가.

> 비즈니스 정의 확정 필요. `docs/db/검사자리스트-월별건수.md` 참조 후 결정.

### 🟠 P2. 월간고객지원 (intake-conversion) — 데이터 출처 다름

**위치:** `backend/src/main/java/com/bviit/analytics/repository/stats/IntakeConversionStatsRepository.java`

**문제:**
- Excel "월간고객지원" 시트는 헤더에 "**이아이씨앤 홈페이지 - 카테고리별 통계**" 명시 → **외부 콜센터 시스템(EICN)** 데이터로 추정
- API는 **SOFTCRM 운영 DB** 기반

두 데이터 출처의 분류 체계와 수집 기준이 처음부터 다름. 따라서 일치 자체가 불가능한 항목입니다.

**해결 옵션:**

- **옵션 A (권장)** — Excel "월간고객지원"은 EICN 별도 데이터로 인정하고 비교 대상에서 제외. 화면 라벨도 "콜 인입(SOFTCRM 기준)"으로 명시.
- **옵션 B** — EICN 시스템과 연동 또는 데이터 수입 경로 설계 (장기 과제).

> 비즈니스 담당자에게 어느 쪽이 "공식 지표"인지 확인 필요.
