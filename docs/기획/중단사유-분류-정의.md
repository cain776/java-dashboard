# 중단 사유 분류 — 정의 및 수정 가이드

> 대상 차트: [`/report/monthly#c-stop-reason`](../../frontend/src/pages/report/MonthlyReportPage.tsx) (월간 레포트 #23) · [`/stats/stop-reason`](../../frontend/src/pages/consultation/StopReasonPage.tsx) (전용 페이지)
> 최종 갱신: 2026-07-15 · 기준 커밋: `fd54e30`

## 한 줄 요약

검사 중단(`EXAM.STOP_YN='Y'`) 건을 **상담사가 드롭다운에서 고른 정형 코드 `EXAM.CANCEL_CD`** 로 7개 사유 버킷에 분류한다. **분류 규칙은 SQL 한 곳**([find-stop-reason-monthly.sql](../../backend/src/main/resources/sql/consultation/find-stop-reason-monthly.sql))에 있다.

## 이력 — 왜 지금 방식인가

| 시점 | 방식 | 결과 |
|------|------|------|
| ~2026-07-14 | `EXAM_MEMO` 자유텍스트 키워드 LIKE 추정 | ❌ 수술권유X 과소·기타 과다로 골든과 어긋남 (4월: 레거시 수술권유X 45 ↔ 우리 recommendX 25 + other 25) |
| 2026-07-14~ (`fd54e30`) | `EXAM.CANCEL_CD` 정형코드 | ✅ 2026-06 골든과 **전 항목 완전 일치** (38 = 21·3·0·0·3·7·4) |

> ⚠️ **"사유 코드가 DB에 없다"는 옛 전제는 틀렸다.** `EXAM.CANCEL_CD`(char(3)) + 마스터 `CANCEL_CFG`가 존재한다. 메모나 오래된 주석에 이 문장이 남아 있으면 그게 낡은 것이다 — **SQL을 열어서 확인할 것.**

## 데이터 흐름

```
EXAM.CANCEL_CD (char(3), 상담사 드롭다운 선택)
   │  마스터: CANCEL_CFG (CANCEL_CD → CANCEL_REASON)
   ▼

   ▼

   ▼

```

## 검증하는 법

```sql
-- 1) 특정 월의 CANCEL_CD 원본 분포 — 미매핑 코드가 other로 새는지 확인
SELECT LTRIM(RTRIM(ISNULL(e.CANCEL_CD,''))) AS cd, COUNT(*) AS cnt
FROM EXAM e WITH(NOLOCK)
WHERE e.STOP_YN='Y' AND e.EXAM_DATE BETWEEN '2026-06-01' AND '2026-06-30'
GROUP BY LTRIM(RTRIM(ISNULL(e.CANCEL_CD,''))) ORDER BY cnt DESC;

-- 2) 코드 마스터 라벨 확인
SELECT * FROM CANCEL_CFG WITH(NOLOCK) ORDER BY CANCEL_CD;
```

**합격 기준**
- 버킷 합 = 해당 월 `STOP_YN='Y'` 건수 = 종합표 `중단수`
- 골든(월간보고 PDF) 중단사유 차트와 **사유별로** 대조 — 합계만 맞고 분해가 갈리면 매핑이 틀린 것 (구 EXAM_MEMO 방식이 딱 이 증상이었다)
- 2026-06 기준 정답: **수술권유X 21 · 렌즈삽입불가 3 · 원추각막 0 · 아벨리노 0 · 녹내장 3 · 시력변화 7 · 기타 4 = 38**

## 함정 (Gotchas)

1. **`other`는 잔차다.** 매핑에 없는 코드·빈 코드는 **조용히 `other`로 떨어진다.** 새 `CANCEL_CD`가 운영에서 생겨도 에러 없이 기타만 늘어나므로 **티가 안 난다.** → 골든 대조에서 기타가 부풀면 이걸 먼저 의심할 것. (위 검증쿼리 1번)
2. **`CANCEL_CD`는 이름과 달리 "취소 플래그"가 아니라 "중단 사유 코드"다.** 그런데 다른 쿼리들은 같은 컬럼을 **취소 판정**에 쓴다 — [지표정의.md](../db/지표정의.md) §검사수 제외조건의 `AND ISNULL(e.CANCEL_CD,'') = ''`. **같은 컬럼, 두 용도.** 한쪽 의미로만 알고 고치면 다른 쪽이 깨진다.
3. **EXAM은 고객당 1행 덮어쓰기** → 과거월 중단건수·사유는 **소급 변동**한다. 1월 중단 환자가 3월 재검사하면 1월 중단건수가 줄어든다. 같은 달을 다른 날 조회하면 값이 달라질 수 있음 ([EXAM.md](../db/tables/EXAM.md) §함정, 메모 `exam-overwrite-stop-yn`). 정확한 고정은 마감 스냅샷 필요.
4. **`STOP_YN` NULL 비중이 큼**(PROD: N=353,784 / NULL=34,712 / Y=27,005) → `STOP_YN='Y'`와 `STOP_YN<>'Y'`는 다르다.
5. **`CANCEL_CFG` 테이블 문서가 없다** (`docs/db/tables/`에 미등재). 전체 코드 목록은 위 검증쿼리 2번으로 직접 확인해야 한다. 본 문서 매핑표의 코드는 `fd54e30` 당시 검증분이라 **전수가 아닐 수 있다.**

## 미확인

- **`CANCEL_CFG` 전체 코드 목록 미확보** — 2026-07-15 mssql-prod·work 양쪽 TCP 1433 접속 불가로 재확인 못 함. 위 매핑표는 `fd54e30` 커밋 당시 검증된 범위(2026-06 prod 기준 완전 일치)이며, 그 달에 등장하지 않은 코드는 반영돼 있지 않을 수 있다.
- 접속 복구되면 검증쿼리 2번으로 전수 확인 후 본 문서 매핑표를 갱신할 것.
