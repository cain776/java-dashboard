## 정의 (스냅샷 일별 필드 → 월합산)

- **콜** = 시력교정(`callReservation` + `tmReservation` + `tmRecounselReservation`) + 백내장(`newPatient`(★신환) + `tmReservation`)
- **온라인** = 시력교정(`homeReservation` + `naverReservation` + `kakaoReservation`) + 백내장(`kakaoCataractReservation` + `kakaoPresbyopiaReservation`(=0) + `onlineReservation`)
- **종합** = 콜 + 온라인 → 스냅샷 총예약(시력교정 6채널 + 백내장 총)과 **검산 일치**(6개월 전부)

## 결정 배경

- **콜**: RSS 합이 레거시 월간보고값과 2·3·4월 **단위까지 정확 일치**(1월만 1,476 vs 레거시 1,432, +44) → 정의 확증.
- **온라인**: RSS(538·504)가 레거시(493·465)보다 **일관되게 +40~79 높음**. 원인은 **네이버 하나** — RSS 네이버(4월 300)가 라이브 `RESERVE_PATH='NAVER'`(225)보다 과다. 홈·카톡은 잘 맞음. 네이버·카톡은 CRM 밖 다중소스라 원래 재현 불가.
- 레거시 월간보고는 **담당자 2명이 서로 다르게 집계**해 비일관 → 우리가 직접 보는 작업자 PDF(=RSS 스냅샷)로 단일화.

> ⚠️ 레거시값(온라인 493·465, 종합 2,425 등)으로 되돌리지 말 것. 코드에도 동일 경고 주석 있음.
