Overview

Summary

비앤빛 Analytics KPI 대시보드

Product Goals

비앤빛 안과 병원 경영진이 핵심 성과 지표(KPI)를 한눈에 파악하고, 현장의 수기 및 엑셀 기반 비효율적인 데이터 집계 방식을 시스템화하여 데이터 기반의 의사결정을 강화하도록 지원합니다.

Context

현재 비앤빛 안과 내부에서 수기 및 엑셀로 관리되던 주요 운영 데이터를 시스템화하고 시각화하기 위해 개발 중인 플랫폼입니다. 5종의 통계 API는 연결이 완료되었으며, 나머지 기능은 설계 단계에 있습니다.

Problem & Solution

User Problem

병원 경영진은 중요한 핵심 성과 지표(KPI)를 수동으로 집계하고 분석해야 하는 비효율성을 겪고 있으며, 정확하고 신속한 의사결정을 위한 통합된 데이터 지표가 부족합니다.

Solution

비앤빛 Analytics 대시보드는 병원 운영의 핵심 KPI를 실시간으로 시각화하여 제공하고, 수기 및 엑셀 작업을 대체하여 데이터 집계의 정확성과 효율성을 높입니다. 이를 통해 경영진은 직관적인 데이터 인사이트를 얻을 수 있습니다.

Differentiator

일반적인 BI 툴과 달리, 비앤빛 안과에 특화된 핵심 성과 지표와 운영 데이터를 맞춤형으로 제공하여 병원 현장의 특수성을 반영합니다. 기존의 분산된 수동 데이터 관리 방식에서 벗어나 통합적이고 자동화된 데이터 분석 환경을 구축하여 데이터 기반의 전략 수립을 지원합니다.

Target & Scenario

Target Users

비앤빛 안과의 경영진 및 관리자. 이들은 병원 운영의 전반적인 성과를 파악하고, 효율적인 자원 배분 및 전략적 의사결정을 담당합니다. 정기적으로 병원 실적을 검토하고 개선점을 찾아야 하는 역할을 수행합니다.

User Scenarios

주간 경영 회의를 앞둔 병원 관리자 '김팀장'은 지난주 병원 운영 성과를 파악하기 위해 비앤빛 Analytics 대시보드에 접속합니다. 대시보드에서 수술 건수, 매출, 예약, 신규 환자 등 핵심 KPI를 한 화면에서 확인하고, 월별 추이 변화를 시각화된 차트를 통해 빠르게 분석합니다. 이 데이터를 바탕으로 회의에 필요한 보고서를 준비하고, 개선이 필요한 영역을 파악하여 다음 주 계획을 수립합니다.

Success & Risks

Key Metrics

대시보드 월간 활성 사용자(MAU) 수 및 접속 빈도 (사용자 채택률), 대시보드 리포트 생성 또는 데이터 조회 시간 단축 (효율성), 수기/엑셀 기반 리포트 생성 비율 감소 (자동화 대체율), 데이터 기반 의사결정으로 인한 병원 운영 효율성 증대 (간접 지표).

Risks

MSSQL 읽기 전용 정책으로 인한 데이터 동기화 및 실시간성 제약, 기존 레거시 시스템과의 데이터 연동 복잡성 및 정확성 문제, 사용자(경영진)의 새로운 시스템 적응 및 학습 저항, 데이터 보안 및 접근 권한 관리의 중요성.
