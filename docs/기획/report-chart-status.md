## 주의 (데이터 신뢰성)

- **EXAM·Cataract_Exam은 고객당 1행 덮어쓰기 스냅샷** → 과거월 수치 사후 변동 (지표정의 §1.9.1.1, 메모 `exam-overwrite-stop-yn`). 레포트는 **마감 스냅샷** 적용 필요([db/지표-마감스냅샷-기획.md](../db/지표-마감스냅샷-기획.md)).
- **예약 콜·온라인**은 콜센터 MySQL·등록일 다중소스라 MSSQL 단독 재현 불가 → **2026값은 추정 라이브** (지표정의 §5.1, 메모 `reservation-overall-not-reproducible`).
- 레거시 대비 검증 구간(**2026-07-15 갱신**): **1~4월**(4월 PDF) + **6월**(6월 PDF, [§레거시 검증 — 2026년 6월])이 대조 완료. **5월**은 6월 PDF에 골든이 있으나 대조 미실시. 검사·중단 계열은 덮어쓰기 스냅샷 특성상 재조회 시점에 따라 수치가 소급 변동될 수 있음 — 2026-06-22 재실측 시 #11·12·15·25는 ±2~4건 변동, 평균차는 불변. 세그먼트·외래·수술 계열은 0 드리프트.
- **화면 표시값 ≠ 라이브값**인 구간이 있다. `MONTHLY_REPORT_PDF_2026_CHARTS` 시드가 우선하므로 검증 시 **API 응답이 아니라 시드 적용 후 값**을 봐야 한다 — 2026 기준 예약 3종은 1~6월, 나머지 라이브 계열은 1~4월이 시드 고정 구간.
