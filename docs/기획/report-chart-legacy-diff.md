## 해석 — 차이의 원인

- **#24·25 백내장/시력교정 수술 = 0건 (완전 일치)** → 수술 환자수 집계는 레거시와 동일 로직.
- **#26 총 수술수 ~26건 차이**: 대시보드 `total = 시력교정 + 백내장 환자수`만 합산. 레거시 총수술수는 **드림렌즈·기타 수술(월 약 25건)**을 포함 → 구조적 정의 차이(버그 아님). 레거시와 맞추려면 기타 수술 합산 필요.
- **#15 일반 검사 — 정의 주의**: 레거시 `일반검사 = 시력교정검사 − 원데이검사`. 기존 status 문서의 `examCount − oneDay` 정의를 쓰면 **월 ~140건** 오차(총검사 기준이라 백내장 세션 포함). `visionCorrection − oneDay`로 파생하면 **~4건**으로 수렴 → 파생식 정정 권장.
- **#23 중단 사유**: 월 합계는 일치(4월 65=65)하나 **사유 분류 매핑이 상이**(레거시 "수술권유X" 45 ↔ 대시보드 `recommendX` 25 + `other` 25). 7종 매핑표 재확인 필요.
- **#1~3 예약(예약종합·콜·온라인)**: 콜센터 MySQL·등록일 다중소스로 **MSSQL 단독 재현 불가 → 2026은 추정 라이브**. 그럼에도 월 평균 14~18건(3월 콜 46건이 최대) 수준으로 근접. 추정 보정 여지.
- **#11·13·14·28** 등 검사·외래 계열은 월 3~6건 차이로 **EXAM 1행 덮어쓰기 스냅샷의 사후 변동 범위** 내. 마감 스냅샷 적용 시 더 수렴 예상.
- **비율(#17·18·21·22)**: 1~3%p 차이로 모두 근접. 예약률은 검사자 분모 기준월 차이에서 소폭 발생.

## 미완성 10종 (차이 산출 불가)

- **검사 유입 7종**(#4~10): 월별 trend API 없음. (주간 API `overall-exam/weekly`에는 `introGeneral/Customer/Staff`, `jobOffice/Student/Etc` 필드 존재 → 월 집계 API 신설 시 비교 가능). #7 B2B 군인·기업은 분류 자체가 신규.
- **세그먼트 비율 3종**(#16·19·20): 일반검사 비율 / 시력교정 일반예약률 / 원데이 예약률 — 원데이·일반 분모/분자 trend 미제공.

> ⚠️ 본 비교는 **6월 진행중(부분월) 제외, 1~4월 확정 구간**만 사용. 검사·중단 계열은 덮어쓰기 스냅샷 특성상 재조회 시점에 따라 수치가 소급 변동될 수 있음.
