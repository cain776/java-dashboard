## 2. 섹션별 상세

### 2.1 유입(검사예약) — 3W~8W

구현 완료된 페이지: 예약 건수, 검사 건수, 상담 전환율

| 주차 | 활동 | 산출물 |
|------|------|--------|
| 3W | 지표 계산 로직 구현 | 예약/검사/상담 쿼리, Service 클래스 |
| 4W | 데이터 파이프라인 구축 | Repository → Controller API 연결 |
| 5W | 대시보드 UX 설계 | KPI 카드 + 차트 레이아웃 기획 |
| 6W | 시각화 개발 | Recharts 기반 프론트 페이지 구현 |
| 7W | 데이터 검증 및 QA | 수기 대장 vs 시스템 수치 비교 검증 |
| 8W | 운영 전환 | 리얼 DB 연결, 현장 배포 |

**현재 구현된 API**:
- `GET /api/stats/reservation` — 예약 건수
- `GET /api/stats/reservation/monthly` — 예약 월별 추이
- `GET /api/stats/examination/monthly` — 검사 월별 추이
- `GET /api/stats/consultation-rate` — 상담 전환율
- `GET /api/stats/intake-conversion` — 유입 전환 통합

**현재 구현된 패널 API** (KPI/트렌드/구성비):
- `/api/stats/reservation/kpi`, `/trend`, `/composition`
- `/api/stats/examination/kpi`, `/trend`, `/composition`
- `/api/stats/consultation-rate/kpi`, `/trend`, `/composition`

### 2.2 수술 — 3W~8W

구현 완료된 페이지: 수술 건수, 주요 수술별 비중

| 주차 | 활동 | 산출물 |
|------|------|--------|
| 3W | 지표 계산 로직 구현 | 수술 건수/비중 쿼리 |
| 4W | 데이터 파이프라인 구축 | Repository → Controller API |
| 5W | 대시보드 UX 설계 | 수술 통계 레이아웃 |
| 6W | 시각화 개발 | 수술 건수/비중 차트 |
| 7W | 데이터 검증 및 QA | 수술 데이터 정합성 확인 |
| 8W | 운영 전환 | 리얼 DB 연결 |

**현재 구현된 API**:
- `GET /api/stats/surgery/monthly` — 수술 월별 추이
- `GET /api/stats/surgery-ratio` — 수술별 비중
- `/api/stats/surgery/kpi`, `/panel/trend`, `/panel/composition`

### 2.3 해외 환자 관련 지표 — 1W~12W

| 주차 | 활동 | 산출물 |
|------|------|--------|
| 1W | 지표 정의 정제 | 해외 환자 지표 정의서 (국적별, 수술유형별) |
| 2W | 비앤빛 실무 협의 | 현장 요구사항 확정, 데이터 출처 합의 |
| 5W | 지표 계산 로직 구현 | 해외 환자 필터링 쿼리, 국적별 집계 |
| 6W | 데이터 파이프라인 구축 | Repository → Service → Controller |
| 7W | 대시보드 UX 설계 | 해외 환자 대시보드 와이어프레임 |
| 8W | 시각화 개발 | 국적별 차트, 추이 그래프 |
| 10W | 데이터 검증 및 QA | 수기 데이터 대조 |
| 12W | 운영 전환 | 배포 및 현장 교육 |

**선행 과제**:
- 해외 환자 판별 기준 확정 (국적? 여권? VISA 유형?)
- 해외 환자 관련 테이블/컬럼 매핑 확인

## 3. 구현 프로세스 (각 지표 공통)

```
1W-2W           3W-4W              5W-6W             7W              8W
┌──────┐    ┌──────────┐     ┌──────────┐    ┌──────────┐    ┌──────────┐
│지표정의│ →  │계산 로직  │  →  │UX 설계   │ →  │데이터     │ →  │운영 전환  │
│실무협의│    │파이프라인 │     │시각화 개발│    │검증 & QA │    │          │
└──────┘    └──────────┘     └──────────┘    └──────────┘    └──────────┘
 기획 100%    백엔드 중심       프론트 중심      기획+개발 협업    전체
```

### 각 단계 상세

| 단계 | 담당 | 핵심 활동 |
|------|------|----------|
| 지표 정의 정제 | 기획팀 | 지표 산식, 집계 주기, 데이터 출처 확정 |
| 비앤빛 실무 협의 | 기획팀 + 현장 | 현장 요구사항 수집, 용어 통일 |
| 지표 계산 로직 구현 | 개발 | Entity/DTO, Repository(SQL), Service |
| 데이터 파이프라인 구축 | 개발 | Controller API, MSSQL 연동 검증 |
| 대시보드 UX 설계 | 기획팀 | 와이어프레임, KPI 카드/차트 구성 |
| 시각화 개발 | 개발 | React 페이지, Recharts 차트, API 연동 |
| 데이터 검증 및 QA | 기획+개발 | 수기 대장 vs 시스템 수치 비교, 엣지케이스 |
| 운영 전환 | 전체 | 리얼 DB 전환, 현장 교육, 모니터링 |
